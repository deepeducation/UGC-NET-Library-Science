👥उपयोगकर्ता के प्रकार (Types of Users)

Library Users को विभिन्न आधारों पर वर्गीकृत किया जाता है। UGC NET में इनकी विशेषताएँ और Information Needs पूछी जाती हैं।

🎓
Academic Users
शैक्षणिक उपयोगकर्ता
  • Students — UG, PG, PhD
  • Faculty — Teachers, Researchers
  • Research Scholars
  • Needs: Journals, Theses, Textbooks
  • Use: OPAC, Database, e-Resources
  • Peak Use: Exam time, Research
🏘️
Public Library Users
सार्वजनिक पुस्तकालय उपयोगकर्ता
  • General Public — सभी आयु
  • Children, Senior Citizens
  • Job Seekers, Homemakers
  • Needs: Fiction, News, Govt. Info
  • Recreational + Informational
  • Heterogeneous Background
🔬
Special Library Users
विशेष पुस्तकालय उपयोगकर्ता
  • Scientists & Researchers
  • Engineers, Doctors, Lawyers
  • Corporate Professionals
  • Needs: Current, Specific, In-depth
  • Use: CAS, SDI, Patents, Standards
  • High Expertise Level
🏛️
National Library Users
राष्ट्रीय पुस्तकालय उपयोगकर्ता
  • Advanced Scholars
  • Bibliographers, Historians
  • Archive Researchers
  • Needs: Rare Books, Archives
  • Reference Only Services
  • Scholarly Research Focus
💻
Digital/Virtual Users

  • Remote Users — Online Access
  • E-journal, E-book Users
  • Database Searchers

  • Use: VPN, Shibboleth, OpenURL

🎯Information Needs (सूचना आवश्यकताएँ)

🎯

Information Needs — अवधारणा और प्रकार

Robert Taylor (1968) — 4 Levels of Information Need | Visceral · Conscious · Formalized · Compromised
Robert Taylor 1968

Robert Taylor (1968) ने Information Needs को 4 Levels में वर्गीकृत किया — यह UGC NET में बारंबार पूछा जाता है।

Q1 — Visceral Need
  • Vague, Undefined Need
  • User को खुद नहीं पता
  • "कुछ चाहिए" — Gut Feeling
  • Sub-conscious Level
Q2 — Conscious Need
  • Mental Description
  • User के Mind में है
  • अभी व्यक्त नहीं
  • Ambiguous Stage
Q3 — Formalized Need
  • Clearly Stated
  • Formal Statement में
  • Librarian को बता सकते हैं
  • Rational Expression
Q4 — Compromised Need
  • Modified for System
  • Search Query बनती है
  • Library System के अनुसार
  • Final Search Statement
🎯 UGC NET Memory Trick Robert Taylor के 4 Levels = "V-C-F-C" = Visceral → Conscious → Formalized → Compromised। या याद करें: "आँत → दिमाग → जुबान → Search Box"।

Information Needs के अन्य प्रकार

  • Current Need — Latest Info
  • Retrospective Need — Past Info
  • Exhaustive Need — Complete Info
  • Everyday Need — Routine Info
🎭 Purpose के आधार पर
  • Research Need — In-depth Study

  • Recreational Need — Leisure
📐 Format के आधार पर
  • Factual Need — Specific Fact

  • Directional Need — Guidance

🔬User Studies — परिचय और महत्व

User Studies (उपयोगकर्ता अध्ययन) वह व्यवस्थित शोध है जिसके द्वारा Library Users की Information Needs, Information Seeking Behaviour (ISB) और Information Use Patterns का अध्ययन किया जाता है।

🎯 User Studies के उद्देश्य
  • Users की Information Needs जानना
  • Library Services को User-centric बनाना
  • Collection Development में सहायता
  • Gap Analysis — Needs vs. Available Resources
  • User Satisfaction Measure करना
  • नई Services Design करना
  • Budget Allocation में Priority तय करना
📚 User Studies का विकास
  • 1948 — Royal Society Scientific Information Conference — पहला बड़ा User Study
  • 1958 — International Conference on Scientific Information (Washington)
  • 1965 — INFROSS Study (Bath University)
  • 1970s — Social Sciences User Studies शुरू
  • 1980s — ISB Models विकसित (Wilson, Ellis)
  • 1990s+ — Digital Era User Behaviour

परिभाषा Information Seeking Behaviour (ISB) = वह सम्पूर्ण प्रक्रिया जिसके द्वारा व्यक्ति अपनी Information Needs को पहचानता है, सूचना की खोज करता है, उसका मूल्यांकन करता है और उसका उपयोग करता है। Wilson (1999) के अनुसार ISB = Information Behaviour का एक भाग।

🧭Information Seeking Behaviour (ISB) Models

ISB Models उस process को describe करते हैं जिससे लोग Information Search करते हैं। UGC NET में इन Models के Steps, Authors और Years — सभी पूछे जाते हैं।

🔵Wilson का Information Behaviour Model (1981 & 1996)

🔵

T.D. Wilson — Information Behaviour Model

1981 (First Model) | 1996 (Revised — Nested Model of Information Behaviour)
Wilson 1981 & 1996

T.D. Wilson ने दो प्रमुख Models दिए जो ISB की समग्र Framework प्रदान करते हैं।

Wilson Model 1981 — पहला Model

इस Model में Wilson ने बताया कि Information Need = Physiological, Affective, Cognitive Needs से उत्पन्न होती है। User Information को Seek करता है → Barrier आती है → Information System या अन्य Sources से Seek करता है।

→
Information Needउत्पन्न होती है
→
Information SeekingLibrary / System
→
BarriersPersonal / Interpersonal
→
Information Useया Failure

Wilson Model 1996 — Revised (Nested Model)

1996 में Wilson ने Nested Model प्रस्तुत किया। इसमें Information Behaviour के 3 Levels हैं जो एक-दूसरे में Nested (शामिल) हैं:

🔵 Level 1 — Information Behaviour
  • Broadest Level
  • सभी Behaviours जो Information से सम्बन्धित
  • Passive Attention भी शामिल
  • Information Need, Seek, Use सभी
🟡 Level 2 — ISB (Seeking)
  • Information Seeking Behaviour
  • Purposive Information Seeking
  • Active Search — Internet, Library
  • Wilson (1999) = "purposive seeking of information"
🟢 Level 3 — Search Behaviour
  • Most Specific Level
  • Query Formulation
  • Database Search, OPAC Search
  • Interaction with System

Barriers — Wilson 1981 Wilson ने 3 प्रकार की Barriers बताईं: (1) Personal Barriers — Time, Education, Language। (2) Interpersonal Barriers — Fear of asking। (3) Environmental Barriers — Distance, Cost, Access।

🎯 UGC NET Key Fact Wilson का 1996 Model = Nested Model। Information Behaviour (Outer) ⊃ Information Seeking Behaviour (Middle) ⊃ Information Search Behaviour (Inner)। Intervening Variables: Stress Coping, Risk/Reward, Self-Efficacy, Perception of Source।

🟢Ellis का Information Seeking Model (1989)

🟢

David Ellis — Behavioural Model of ISB

1989 (Social Scientists Study) | 1993 (Scientists) | 6 Core Activities
Ellis 1989

David Ellis ने Social Scientists का Interview करके 6 Core Activities Identify कीं। बाद में Ellis, Cox & Hall (1993) ने Scientists & Engineers पर Study की।

Ellis के 6 Steps — Behavioural Model
1
Starting — प्रारम्भिक जानकारी Identify करना। Initial Sources — Colleagues, Known Journals, Reviews से शुरुआत। "Seed References" तलाश।
2
Chaining — References को Forward या Backward follow करना। Forward Chaining = किसी Paper को Cite करने वाले Papers। Backward Chaining = References List को follow करना।
3
Browsing — Semi-directed Searching। Journals, Shelves, Databases में Scan करना। Serendipitous Discovery।
4
Differentiating — Sources की Quality, Relevance, Reliability के आधार पर Filter करना। Known Sources vs. Unknown Sources।
5
Monitoring — Current Awareness बनाए रखना। Core Sources को Regular Check करना। CAS/SDI जैसी Services का उपयोग।
6
Extracting — Select Source से Relevant Information निकालना। Systematic Extraction। Note-taking, Downloading।

Note Ellis Model को बाद में Wilson (1999) ने अपने Framework में शामिल किया। Ellis के 1993 Study में Verifying (Accuracy Check) और Ending (Search Conclusion) भी जोड़े गए — Total 8 Activities।

🎯 Memory Trick — Ellis के 6 Steps "S-C-B-D-M-E" = Starting → Chaining → Browsing → Differentiating → Monitoring → Extracting। याद करें: "Start Chain Browse Diff Monitor Extract"

🔮Dervin का Sense-Making Theory (1983)

🔮

Brenda Dervin — Sense-Making Theory

1983 | User-centric Approach | Time-Line · Gap · Use (Outcome)
Dervin 1983

Brenda Dervin का Sense-Making Theory एक User-centric, Constructivist approach है। इसमें Information = वह नहीं जो System में है, बल्कि वह है जो User अपने Context में Create करता है।

🔑 Dervin के Key Concepts
  • Sense-Making — User खुद Meaning बनाता है
  • Gap — Information Need का मेटाफर
  • Bridge — Gap को भरने का तरीका
  • Outcome — Information Use का Result
  • Information = Subjective, Context-dependent
📊 Wilson vs Dervin vs Ellis
  • Wilson = Macro Level Framework; Barriers; Nested Model
  • Ellis = Micro Level; 6 Behavioural Steps; Academic Researchers
  • Dervin = User Constructivist; GAP Metaphor; Sense-Making
  • तीनों = ISB का अलग-अलग Perspective

Kuhlthau — ISP Model (1991)
Information Search Process

  • Affective, Cognitive, Physical dimensions
  • Student Research Process पर आधारित
  • Uncertainty Principle = Beginning में Uncertainty; Confidence बढ़ती जाती है

  • Analytical vs Browsing Strategies
  • OPAC, Web Search Behaviour

Savolainen — ELIS Model (1995)
Everyday Life Information Seeking
  • Non-Work Context — Daily Life
  • Way of Life + Mastery of Life
  • Public Library Users के लिए उपयुक्त
  • Habitual Information Seeking

📚User Education (उपयोगकर्ता शिक्षा)

User Education वह प्रक्रिया है जिसके द्वारा Library के Users को Library Resources, Services और Information Tools का प्रभावी उपयोग करना सिखाया जाता है। इसे Library Instruction या Bibliographic Instruction भी कहते हैं।

परिभाषा ACRL (Association of College and Research Libraries) के अनुसार — User Education का लक्ष्य है Users को Information Literate बनाना — अर्थात्, Information की Need को Identify करना, उसे Locate करना, Evaluate करना और Effectively Use करना।

User Education के Level/प्रकार

📈

User Education — 3 मुख्य Levels

Library Orientation → Bibliographic Instruction → Information Literacy
3 Levels
Level 1 — Library Orientation
  • Basic Introduction to Library
  • Library Tour (Physical / Virtual)
  • Membership, Borrowing Rules
  • Library Layout, Sections
  • OPAC Introduction
  • नए Users के लिए — First Step
Level 2 — Bibliographic Instruction (BI)
  • Information Sources का Knowledge
  • OPAC Advanced Search
  • Reference Sources — Use करना
  • Database Search Skills
  • Citation & Bibliography Writing
  • Academic Users के लिए
Level 3 — Information Literacy (IL)
  • Highest Level
  • ACRL Framework (6 Frames)
  • Big6 Model, SCONUL 7 Pillars
  • Critical Evaluation of Sources
  • Ethical Use of Information
  • Lifelong Learning Skills

✓निष्कर्ष

Library का सम्पूर्ण अस्तित्व User के लिए है। User Studies हमें बताती है कि Users को क्या चाहिए, वे कैसे Information खोजते हैं और Library किन Barriers को दूर करे।

Wilson, Ellis, Dervin और Kuhlthau के Models ने ISB को Scientific Framework दिया। User Education Programs — Library Orientation से लेकर Information Literacy तक — Users को Self-reliant और Critically Thinking बनाते हैं। 21वीं सदी में Digital Users का बढ़ता वर्ग Library Services को Re-design करने का अवसर देता है।

❓FAQs — 10 महत्वपूर्ण प्रश्न

Q1. Robert Taylor ने Information Needs के कौन से 4 Levels बताए?+
Robert Taylor (1968) के अनुसार Information Need के 4 Levels: (1) Visceral Need (Q1) — Vague, Sub-conscious, User को खुद नहीं पता। (2) Conscious Need (Q2) — Mental Description, अभी Expressed नहीं। (3) Formalized Need (Q3) — Clearly Stated, Librarian को बता सकते हैं। (4) Compromised Need (Q4) — System के लिए Modified Search Query बनती है। Memory Trick: V-C-F-C।
Q2. Wilson का Information Behaviour Model (1996) क्या है?+
Wilson का 1996 Revised Model = Nested Model। तीन Levels Nested हैं: (1) Information Behaviour (Outermost) — सभी Passive और Active behaviours। (2) Information Seeking Behaviour (Middle) — Purposive ISB। (3) Information Search Behaviour (Innermost) — Query Formulation, Database Interaction। Intervening Variables: Stress Coping, Self-Efficacy, Risk/Reward, Perception of Source।
Q3. Ellis Model के 6 Steps क्या हैं? Memory Trick बताएँ।+
Ellis Model (1989) के 6 Steps: (1) Starting — Initial Sources; (2) Chaining — Forward/Backward Reference Chaining; (3) Browsing — Semi-directed Scanning; (4) Differentiating — Quality/Relevance Filter; (5) Monitoring — Current Awareness; (6) Extracting — Systematic Extraction। Memory: "Start Chain Browse Diff Monitor Extract" = S-C-B-D-M-E। 1993 में Verifying और Ending भी जोड़े गए।
Q4. Dervin का Sense-Making Theory क्या है?+
Brenda Dervin (1983) का Sense-Making Theory एक Constructivist Approach है। मूल Framework: Situation (Context) → GAP (Information Need) → Bridge (Seeking) → Outcome/Use (Sense Made)। Key Idea: Information = वह नहीं जो System में है; बल्कि User अपने Context में जो Meaning बनाता है। GAP = Metaphor for Information Need। यह User-centric approach है — Public Library Users के लिए विशेष रूप से उपयुक्त।
Q5. Kuhlthau का ISP Model क्या है? Uncertainty Principle क्या है?+
Carol Kuhlthau (1991) का Information Search Process (ISP) Model में 6 Stages हैं: (1) Initiation, (2) Selection, (3) Exploration, (4) Formulation, (5) Collection, (6) Presentation। हर Stage में Affective (Feeling), Cognitive (Thought), Physical (Action) dimensions हैं। Uncertainty Principle = Process की शुरुआत में Uncertainty/Confusion सबसे अधिक होती है; Formulation Stage के बाद Clarity और Confidence बढ़ती है।
Q6. User Studies क्यों जरूरी है? इसके उद्देश्य क्या हैं?+
User Studies Library के लिए इसलिए जरूरी है: (1) User की Information Needs जानना — क्या चाहिए? (2) Collection Development — कौन सी Books/Journals खरीदें? (3) Service Design — Users की जरूरत के अनुसार नई Services। (4) Gap Analysis — Library क्या Provide करती है vs. User को क्या चाहिए। (5) Budget Priority — कहाँ पैसा लगाएँ? (6) User Satisfaction Measure करना। बिना User Study = Library = User-irrelevant बन जाती है।
Q7. User Education के तीन Levels क्या हैं?+
User Education के 3 Levels: (1) Library Orientation (Level 1) — Library Tour, Membership, OPAC Basics, Layout — नए Users के लिए। (2) Bibliographic Instruction (Level 2) — Reference Sources, Database Search, Citation Writing, OPAC Advanced Search — UG/PG Students के लिए। (3) Information Literacy (Level 3) — ACRL Framework, Big6, SCONUL 7 Pillars, Critical Evaluation, Ethical Use — Researchers के लिए। Highest Level = Information Literate Person।
Q8. Actual Users और Potential Users में क्या अंतर है?+
Actual Users = वे लोग जो वर्तमान में Library का उपयोग कर रहे हैं — Registered Members, Walk-in Users। Potential Users = वे लोग जो Library का उपयोग कर सकते हैं लेकिन नहीं कर रहे — Eligible लेकिन Unregistered। Non-Users = जिन्हें Library की जरूरत नहीं या Awareness नहीं। Library का लक्ष्य = Potential Users को Actual Users में Convert करना। User Studies से Potential Users को Identify किया जाता है।
Q9. Transaction Log Analysis क्या है? इसके क्या लाभ हैं?+
Transaction Log Analysis (TLA) — OPAC, Database, Website के Server Logs का Analysis। Users ने क्या Search किया, कब किया, कितने समय तक — सब Automatically Record होता है। लाभ: (1) Unobtrusive — Users को पता नहीं; (2) Large Sample Size; (3) Objective Data; (4) Longitudinal Study संभव; (5) Cost-effective। सीमाएँ: केवल Behaviour Capture होता है, Motivation नहीं; Robot/Crawler Traffic भी शामिल।
Q10. Forward Chaining और Backward Chaining में क्या अंतर है?+
यह Ellis Model के Chaining Step का Part है: Forward Chaining = किसी Known Paper/Article को किन-किन Papers ने बाद में Cite किया — आगे की ओर Chaining। उदाहरण: Google Scholar में "Cited by" देखना। Backward Chaining = किसी Paper की Reference List को Follow करना — पीछे की ओर Chaining। उदाहरण: Research Paper की Bibliography से Related Papers ढूँढना। दोनों = Academic Research में सबसे Common ISB।
